### 수업

***메소드 오버로딩(overloading)
-> 이름이 같은 여러개의 메소드를 중복 선언하여
사용의 편리성을 제공
-> 리턴타입보다 파라미터값이 중요함

조건
1) 이름이 동일해야함.
2) 인자의 개수가 달라야 함.
3) 인자의 타입이 달라야 함.
4) 동일한 클래스 파일 안에 있어야 함.
5) 메소드에 접근지정자를 private로 선언하면
   외부 클래스에서 호출 시 해당 메소드를 호출할 수 없다.

자바 main함수
*static함수와 *new선언으로 만든 클래스 메소드가 올 수 있다.

DW 코딩 규칙
변수 생성하는 방법
1. 메소드에 리컨값이 있으면 우선 변수로 받는다.
2. 선언 변수가 행동이 일어나서 결과값이 생길 때.
ex) int x = 10; int y = 10;
    int sum = x + y; // sum으로 데이터를 받는다.
3. 선언한 변수가 정말 필요한지 고민.(재활용이 가능한지?)
4. 지역변수에서만 활용할 건지 전역변수로 활용할 건지 고민.
###메소드 분리하는 방법
1. 함수가 어려우면 메인함수에 코딩
2. 메인함수로 결과가  나오면 함수로 분리할 것
3. 고민 고민 또 고민 다시 고민(고민해야 실력이 늘어남!!!)

***문자메소드


***클래스 = 객체 or object
 자바는 <객체/지향/언어>
 객체란 고유한 특성을 가짐.
 객체들이 모여서 하나의 프로그램이 탄생.

 public class는 해당 클래스 파일에 한번 만
 
 객체
 ex) 마우스, 키보드, 모니터, 본체, 스피커

### 객체 지향 언어 특징 (면접 필)
1. 캡슐화
- 내부를 마치 캡슐처럼 보호 

2. 상속
- 클래스를 부모와 자식으로 두고
부모의 기능을 물려받음

3. 다형성
- 같은 이름의 메소드가 다르게 구현
(메소드 오버로딩, 오버라이딩)
